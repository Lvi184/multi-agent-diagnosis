# 修复说明：解决每次对话必走完整流程问题

## 问题描述

用户每次提问，即便是简单问候如"你好"，系统也会强制走完整个多Agent诊断流程：
```
Intake → Structuring → Scale → MoodAngels → Model Verification → Knowledge → Differential → Risk → Validator → Report
```

导致不必要的资源消耗和用户体验问题。

## 解决方案

### 1. 新增：意图分类器 (`app/agents/intent_classifier.py`)

轻量级规则匹配意图分类器，无需LLM调用，实时响应：

| 意图类型 | 说明 | 是否进入诊断流程 |
|---------|------|-----------------|
| `greeting` | 问候/打招呼（你好、hi、在吗等） | ❌ 否 |
| `thanks` | 感谢（谢谢、多谢等） | ❌ 否 |
| `goodbye` | 道别（再见、拜拜等） | ❌ 否 |
| `question_system` | 询问系统相关（你是谁、能做什么等） | ❌ 否 |
| `question_medical` | 询问医学问题（怎么办、怎么治疗等） | ❌ 否 |
| `chitchat` | 闲聊/无关对话 | ❌ 否 |
| `symptom_report` | 症状报告/寻求诊断 | ✅ 是 |

### 2. 修改：`/api/chat` 接口前置拦截 (`app/api/routes.py`)

在执行完整诊断流程前，先进行意图判断：

```python
# 1. 前置意图判断
intent, confidence = intent_classifier.classify(request.message)

# 2. 检查是否有预填写的量表分数（有分数直接进入诊断）
has_scale_scores = ...

# 3. 非诊断意图，直接回复（除非有量表分数）
if not has_scale_scores and not intent_classifier.should_enter_diagnosis(intent):
    reply = intent_classifier.get_response(intent)
    return ChatResponse(session_id=..., reply=reply, report=None)

# 4. 真正的诊断请求，执行完整流程
...
```

### 3. 更新：响应 Schema (`app/schemas/diagnosis.py`)

- `report` 字段改为可选（`Optional[DiagnosisResponse]`）
- 新增 `intent` 字段，返回意图识别结果

### 4. 更新：前端兼容 (`frontend/index.html`)

- 兼容 `report` 为 `None` 的情况
- 显示意图识别结果，让用户知道为什么跳过了完整流程

## 使用效果

### ✅ 场景1：简单问候
**用户输入**："你好"
**系统行为**：识别为 `greeting`，直接友好回复，不进入诊断
**响应**："你好！我是心理健康辅助筛查助手。如果您有情绪、睡眠等方面的困扰需要评估，都可以告诉我。"

### ✅ 场景2：询问系统
**用户输入**："你是谁"
**系统行为**：识别为 `question_system`，直接回答，不进入诊断
**响应**："我是多Agent心理健康辅助筛查原型。我可以帮您进行心理健康初步筛查评估..."

### ✅ 场景3：感谢
**用户输入**："谢谢"
**系统行为**：识别为 `thanks`，直接回复，不进入诊断
**响应**："不客气！如果后续还有需要，随时可以再找我。"

### ✅ 场景4：症状报告
**用户输入**："最近2个月情绪低落，睡不着，对事情没兴趣，有时觉得不想活"
**系统行为**：识别为 `symptom_report`，正常进入10个Agent完整诊断流程

### ✅ 场景5：带量表分数
**用户输入**：任何内容 + PHQ9/GAD7 分数不为0
**系统行为**：直接进入诊断流程

## 性能优势

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| 问候/闲聊 | 约 5-10 秒（10个LLM调用） | < 10 毫秒（纯规则匹配） |
| 资源消耗 | 大量 Token 消耗 | 0 Token 消耗 |

## 部署方式

1. 重启后端服务即可
2. 前端已兼容，无需额外操作

## 测试覆盖

已测试 19 种场景，全部通过：
- 问候类（4种）
- 感谢类（2种）
- 道别类（2种）
- 系统问题（3种）
- 医学问题（2种）
- 症状报告（4种）
- 闲聊类（2种）

边界情况测试：
- 问候+症状混合 → 进入诊断
- 症状+问题混合 → 医学问题回复
- 空文本/空白字符 → 闲聊回复
- 无意义文本 → 未知意图
