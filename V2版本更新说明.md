# MoodAngels-LangGraph V2 更新说明

## 问题反馈

用户反馈原来的交互方式体验不好：
> "界面右边有填信息的；但系统应该引导我，说最近心情如何，感觉如何，然后问有没测过量表，有的话吧分数填右边或者告诉他，没有的话触发量表评估流程，然后是以主动提问的方式实现对话入口与病史采集"

## 更新内容

### ✅ 更新1：新增意图分类器 - 避免非诊断场景强制走完整流程
**文件**：`app/agents/intent_classifier.py`

**功能**：
- 识别 8 种对话意图
- 问候、感谢、道别、询问系统功能等场景，直接回复，不进入诊断
- 只有真正的症状报告才执行完整的多Agent诊断流程
- 纯规则匹配，<10ms响应，无需调用LLM

**效果对比**：

| 用户输入 | 修复前（V1） | 修复后（V2） |
|---------|-------------|-------------|
| "你好" | 强制走完10个Agent流程，输出"信息不足" | 🚀 立即友好回复，引导描述症状 |
| "谢谢" | 强制走完10个Agent流程 | 🚀 立即回复"不客气" |
| "你是谁" | 强制走完10个Agent流程 | 🚀 立即回复系统说明 |
| "最近心情不好" | 直接执行诊断 | ✅ 进入引导式采集流程 |

---

### ✅ 更新2：引导式多轮对话 - 主动提问，逐步采集
**文件**：`frontend/index.html`（已更新），`app/api/routes.py`

**核心改进**：

1. **默认使用 `/api/dialogue` 接口**（替代旧的 `/api/chat`）
   - 支持会话状态保持
   - 支持多轮对话引导

2. **完整的引导流程**：
   ```
   ① 用户描述症状
   → ② 询问：持续时间？
   → ③ 询问：既往史？
   → ④ 询问：用药史？
   → ⑤ 引导：PHQ-9 抑郁量表（9题）
   → ⑥ 引导：GAD-7 焦虑量表（7题）
   → ⑦ 信息收集完整，自动执行完整诊断
   ```

3. **右侧实时显示已收集信息**：
   - 症状
   - 持续时间
   - 既往史
   - 用药史
   - PHQ-9 分数
   - GAD-7 分数

4. **状态指示器**：
   - 📋 信息采集中
   - 🔍 诊断分析中
   - ✅ 评估完成

### ✅ 更新3：优化引导Agent的初始响应
**文件**：`app/agents/dialogue_agent.py`

**改进**：
- 如果用户只是打招呼，没有提到症状，不会直接问"持续多久了"
- 而是会引导用户描述具体情况：
  > "您好！我是心理健康辅助筛查助手。
  > 
  > 为了更好地帮助您，请简单描述一下您的情况，比如：
  > • 最近的心情怎么样？
  > • 睡眠情况如何？
  > • 有没有什么特别的困扰？"

## 两种模式说明

系统现在支持两种模式，自动切换：

### 🟦 引导式评估（默认）
- **触发条件**：用户描述症状，但没有量表分数
- **流程**：多轮对话采集 → 量表评估 → 自动诊断
- **体验**：像和医生对话一样，逐步回答问题

### 🟩 快速诊断模式
- **触发条件**：用户已有量表分数（PHQ-9/GAD-7）
- **流程**：直接进入完整诊断流程
- **体验**：快速出结果，适合已有评估结果的用户
